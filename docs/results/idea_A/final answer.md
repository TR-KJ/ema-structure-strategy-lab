# Idea A EMA Pullback 最終検証まとめ

## 概要

Idea Aでは、EMA順行配列中のEMA20押し戻りから、EMA20再クロスで再加速を狙う手法を検証した。

目的は、裁量判断をなるべく排除し、SetupとTriggerを機械的に定義できるEMA構造手法として成立するかを確認すること。

## 検証した基本構造

### Long Setup

- EMA20 > EMA50
- EMA50 > EMA200
- close > EMA50
- 過去10本以内に low <= EMA20

### Short Setup

- EMA20 < EMA50
- EMA50 < EMA200
- close < EMA50
- 過去10本以内に high >= EMA20

### 初期Trigger

#### Long

- Long Setup成立中
- close > EMA20
- close[1] <= EMA20[1]
- barstate.isconfirmed

#### Short

- Short Setup成立中
- close < EMA20
- close[1] >= EMA20[1]
- barstate.isconfirmed

### 初期SL/TP

- SL：Trigger確定足を含む直近10本の高値/安値
- TP：Trigger確定足close基準のRR 1.0
- Entry：次足始値約定想定
- 決済：SL/TPのみ
- 反対シグナル決済なし

## 共通Strategy設定

pine initial_capital = 1000000 default_qty_type = strategy.percent_of_equity default_qty_value = 2 commission_type = strategy.commission.percent commission_value = 0.04 slippage = 2 pyramiding = 0 process_orders_on_close = false calc_on_every_tick = false 

## 検証期間

| 項目 | 内容 |
|---|---|
| 時間足 | 1分足 |
| 検証期間 | 2021/01/01〜2025/12/31 |
| HTFフィルター | なし |
| 決済 | SL/TPのみ |

# Strategy v1.2 Entry価格基準TP/SL診断

## 目的

v1.1では、TP/SLをTrigger確定足close基準で計算していた。

Entryは次足始値約定想定のため、TP/SL計算基準と実際のEntry価格にズレがあった。

そこでv1.2では、strategy.position_avg_price 基準でTP/SLを計算する診断を行った。

## 結果

v1.2では未決済ポジションが残る問題を確認した。

Entry後でないと strategy.position_avg_price が確定しないため、EntryとExitを同一Entryブロック内で発注できなかった。

また、実際の約定価格と保存済みSLの位置関係によってRiskが無効になるケースが発生した。

その結果、SL/TPが正しく発注されず、ポジションが長期間未決済として残る挙動を確認した。

## v1.2 評価

v1.2はStrategy実装として不安定と判断。

以降の比較対象から除外する。

今後は、v1 / v1.1と同様に、Trigger確定足時点でSL/TPを固定し、EntryとExitを同一Entryブロック内で発注する方式を基本とする。

# 最終結論

Idea A EMA Pullback 初期型は、現時点では不採用とする。

## 理由

- コストなしではPF 1.007でほぼ損益分岐だが、優位性は非常に薄い
- コストありではPF 0.009まで悪化し、実運用耐性がない
- 最低リスク幅フィルターを追加しても、PF改善は不十分
- Entry価格基準TP/SLはStrategy構造的に不安定
- Triggerを強化しても改善は限定的
- RRを上げてもPF 1.0に近づかない
- 1分足のEMA20再クロス型はノイズとコストの影響を強く受ける

## 得られた知見

- EMA20への押し戻り再クロスだけでは、1分足では優位性が薄い
- 1分足では、平均勝ち・平均負けが小さい構造はコストに非常に弱い
- 最低リスク幅フィルターはDDとトレード数を減らす効果はあるが、優位性そのものは作れない
- Triggerを少し強化するだけでは、ノイズ除去として不十分
- SL/TPはStrategy上、Trigger確定足時点で固定し、EntryとExitを同一Entryブロック内で発注する方式が安定する

## 次の方針

Idea Aはここで一旦区切る。

次は、押し戻り継続型ではなく、相場状態をより限定できる以下の案へ移行する。

- Idea B：EMA収縮 → 拡散ブレイク

Idea Bでは、EMAの収縮状態をSetupとして定義し、収縮後のブレイクとEMA拡散をTriggerとして検証する。
